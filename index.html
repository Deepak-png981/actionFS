ActionFS

The ActionFS library is a Node.js module providing a set of asynchronous file system operations. It's built on top of Node.js's fs.promises module and follows SOLID and OOP principles.

Class: ActionFS

This class encapsulates all the file system operations provided by the library.

Methods

validateArgs(args, schema)

  • Purpose: Validates method arguments against a provided schema.
  • Parameters:
    • args (object): The arguments to validate.
    • schema (object): The schema to validate against.
  • Returns: boolean. True if validation passes, throws an error otherwise.

createDirectory(path, options, [callback])

  • Purpose: Creates a new directory.
  • Parameters:
    • path (string): The path to the directory.
    • options (object): The directory creation options.
    • callback (Function, optional): An optional callback function.
  • Returns: Promise<void>.

  • Purpose: Reads the contents of a file.

    • path (string): The path to the file.
    • options (object): The read options.
    • callback (Function, optional): An optional callback function.
  • Returns: Promise<string>. A promise that resolves with the file content.

createFile(path, content, options, [callback])

  • Purpose: Creates a new file with the specified content.
  • Parameters:

    • options (object): The write options.
    • callback (Function, optional): An optional callback function.
  • Returns: Promise<void>.

updateFile(path, content, options, [callback])

  • Purpose: Updates an existing file with new content, or creates it if it doesn't exist.
  • Parameters:
    • path (string): The path to the file.
    • content (string|Buffer): The new content.
    • options (object): The write options.
    • callback (Function, optional): An optional callback function.
  • Returns: Promise<void>.

deleteFile(path, [callback])

  • Purpose: Deletes a file.
  • Parameters:
    • path (string): The path to the file.
    • callback (Function, optional): An optional callback function.
  • Returns: Promise<void>.

copyFile(srcPath, destPath, [callback])

  • Purpose: Copies a file from one path to another.
  • Parameters:
    • srcPath (string): The path to the source file.
    • destPath (string): The path to the destination file.
    • callback (Function, optional): An optional callback function.
  • Returns: Promise<void>.

moveFile(oldPath, newPath, [callback])

  • Purpose: Moves a file from one path to another. This is effectively a rename operation.
  • Parameters:
    • oldPath (string): The current path of the file.
    • newPath (string): The new path for the file.
    • callback (Function, optional): An optional callback function.
  • Returns: Promise<void>.

renameFile(oldPath, newPath, [callback])

  • Purpose: Renames a file.
  • Parameters:
    • oldPath (string): The current path of the file.
    • newPath (string): The new path (name) for the file.
    • callback (Function, optional): An optional callback function.
  • Returns: Promise<void>.

fileExists(path, [callback])

  • Purpose: Checks if a file or directory exists at the given path.
  • Parameters:
    • path (string): The path to the file or directory.
    • callback (Function, optional): An optional callback function.
  • Returns: Promise<boolean>. A promise that resolves with a boolean value.

createDirectoryStructure(filePath, [callback])

  • Purpose: Creates the directory structure needed for a given file path.
  • Parameters:
    • filePath (string): The file path for which to create the directory structure.
    • callback (Function, optional): An optional callback function.

  • Purpose: Builds a representation of the directory structure.
  • Parameters:

    • callback (Function, optional): An optional callback function.

readDirectory(directoryPath, indent, [callback])

  • Purpose: Reads the directory structure and returns it with designated properties.
  • Parameters:
    • directoryPath (string): The path to the directory to read.
    • indent (number, optional): The indentation level for formatting.
    • callback (Function, optional): An optional callback function.

  • Parameters:

ActionFS User Guide

Welcome to the ActionFS User Guide. This document is intended to help you quickly learn how to use the ActionFS library for handling file system operations in Node.js.

Introduction

ActionFS is an asynchronous file system library built on Node.js's fs.promises module. It simplifies tasks like creating, reading, updating, deleting, copying, and moving files and directories.

Getting Started

Installation

Currently, ActionFS is a custom library and not available on npm. To use it, you need to include it directly in your Node.js project.

  1. Download the action-fs.js (for CommonJS) or action-fs.mjs (for ES Modules) file from the source repository.
  2. Place the file in an appropriate directory within your project.

Importing the Library

Using CommonJS Syntax

const ActionFS = require('./index.js');

Basic Usage

Create an instance of ActionFS to start using the library:

const actionFS = new ActionFS();

Detailed Usage Examples

In this section, we'll go through some common use cases and demonstrate how to use the ActionFS library to accomplish them.

Creating a Directory

To create a new directory:

actionFS.createDirectory('path/to/directory', { recursive: true })
    .then(() => console.log('Directory created successfully'))
    .catch(error => console.error('Error creating directory:', error));

The { recursive: true } option allows creating nested directories.

Reading a File

To read the contents of a file:

actionFS.readFile('path/to/file.txt', { encoding: 'utf-8' })
    .then(content => console.log('File content:', content))
    .catch(error => console.error('Error reading file:', error));

This reads the file with the specified encoding and logs its content.

Creating a File

To create a new file with content:

const content = 'Hello, world!';
actionFS.createFile('path/to/file.txt', content, {})
    .then(() => console.log('File created successfully'))
    .catch(error => console.error('Error creating file:', error));

This creates a file and writes the specified content to it.

Updating a File

To update an existing file or create it if it doesn't exist:

const newContent = 'Updated content';
actionFS.updateFile('path/to/file.txt', newContent, {})
    .then(() => console.log('File updated successfully'))
    .catch(error => console.error('Error updating file:', error));

The method checks if the file exists and either updates it or creates a new file with the given content.

Moving a File

To move (or rename) a file:

actionFS.moveFile('path/to/oldFile.txt', 'path/to/newFile.txt')
    .then(() => console.log('File moved successfully'))
    .catch(error => console.error('Error moving file:', error));

Renaming a File

To rename a file:

actionFS.renameFile('path/to/oldFile.txt', 'path/to/newFile.txt')
    .then(() => console.log('File renamed successfully'))
    .catch(error => console.error('Error renaming file:', error));

Checking File or Directory Existence

To check if a file or directory exists at a given path:

actionFS.fileExists('path/to/fileOrDirectory')
    .then(exists => {
        if (exists) {
            console.log('File or directory exists');
        } else {
            console.log('File or directory does not exist');
        }
    })
    .catch(error => console.error('Error checking existence:', error));

Creating Directory Structure for a File Path

To create the directory structure needed for a given file path:

actionFS.createDirectoryStructure('path/to/a/new/file.txt')
    .then(() => console.log('Directory structure created successfully'))
    .catch(error => console.error('Error creating directory structure:', error));

The createDirectoryStructure method creates the necessary directory structure so that the provided file path can be used.

Building a Directory Structure Representation

To build a representation of the directory structure:

actionFS.build({}, { path: 'path/to/directory', type: 'folder' })
    .then(structure => console.log('Directory structure:', structure))
    .catch(error => console.error('Error building structure:', error));

The build method generates a structured representation of the directory and its contents.

Reading Directory Contents

To read the contents of a directory and its subdirectories:

actionFS.readDirectory('path/to/directory')
    .then(contents => console.log('Directory contents:', contents))
    .catch(error => console.error('Error reading directory:', error));

This method reads the directory structure and returns an array with designated properties for each item (folders and files).

Creating Directory and Files from a Structured Object

To create directories and files based on a structured object:

const structure = [    {        type: 'folder',        name: 'folder1',        children: [            {                type: 'file',                name: 'file1.txt',                children: [                    { type: 'content', line: 'File content line 1' },                    { type: 'content', line: 'File content line 2' }                ]
            }
        ]
    }
];

actionFS.createDirectoryUsingObject(structure, 'path/to/baseDirectory')
    .then(() => console.log('Directories and files created successfully'))
    .catch(error => console.error('Error creating directories and files:', error));

This method allows you to create directories and files based on a structured object representing the desired directory structure.

ActionFS Developer Documentation

Welcome to the ActionFS Developer Guide. This document aims to provide developers with the information necessary to understand, extend, and debug the ActionFS library.

Introduction

ActionFS is an asynchronous file system library for Node.js, designed to simplify file system operations. It's built on top of the Node.js fs.promises module and adheres to SOLID and OOP principles.

Library Architecture

The ActionFS library is composed of a single main class, ActionFS, which encapsulates all file system operations. It utilizes static and instance methods to perform various tasks.

Class Structure

  • Class Name: ActionFS
  • Purpose: To provide asynchronous file system operations.
  • Methods: Each method in the ActionFS class is designed to perform a specific file system operation such as reading, writing, copying, or deleting files and directories.

Key Components

  1. Argument Validation: Each method uses the validateArgs static method to validate its arguments against a predefined schema.
  2. Error Handling: Proper error handling is implemented to ensure reliability and ease of debugging.
  3. Asynchronous Operations: All file system operations are asynchronous, using Promises.

Extending the Library

To add new functionality to the ActionFS library:

  1. Create a New Method: Methods should be created within the ActionFS class. Ensure that they follow the existing naming and coding conventions.
  2. Argument Validation: Add argument validation schemas for new methods.
  3. Documentation: Update both user and developer documentation to reflect the new changes.
